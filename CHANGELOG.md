### Fixed

- **TUI ‚Äî `#mode-toggle` slider: `$success` + defensive `.-on` paired rule** ‚Äî Removed the broad `ModeProviderBar Switch > .switch--slider { color: $accent; }` rule (redundant for provider switches, which stock `Switch.-on` already outranks) and replaced the id-only `#mode-toggle` override with paired selectors `ModeProviderBar #mode-toggle > .switch--slider` and `ModeProviderBar #mode-toggle.-on > .switch--slider`, both `color: $success`, so TV (off) and Film (on) both paint the thumb in Textual's **`$success`** token on every built-in theme (including `:light` / solarized-light). The paired `.-on` selector is defensive against future nested Textual variants; the id remains dominant over stock **(0, *, *)** rules. Pinned by `tests/unit/test_mode_provider.py::test_mode_toggle_slider_renders_success_in_both_tv_and_film` (parametrized **`solarized-light`** and **`textual-dark`**, asserts against `app.current_theme.success`). Application **0.8.1** (PATCH); `tui/` module spec **1.7.1**; spec **1.12.1** unchanged.

- **TUI ‚Äî TV/Film `#mode-toggle` slider color parity on `solarized-light`** ‚Äî Stock Textual `Switch` CSS applies `Switch.-on .switch--slider { color: $success; }` and, on light themes, `Switch:light.-on .switch--slider` at specificity **(0, 3, 1)**, which outranked our **(0, 1, 2)** rule `ModeProviderBar Switch > .switch--slider { color: $accent; }`, so Film (on) and TV (off) diverged on solarized-light. **Id-scoped** rules on **`#mode-toggle`** ensure both positions resolve to **`$success`** (Textual's stock `.-on` thumb color) on all built-in themes. Pinned by `tests/unit/test_mode_provider.py::test_mode_toggle_slider_renders_success_in_both_tv_and_film` (parametrized light/dark themes; asserts against `app.current_theme.success`). Application **0.7.2** (PATCH); `tui/` module spec **1.6.2**; spec **1.12.1** unchanged.

- **TUI ‚Äî `ModeProviderBar` shared toggle visual polish** ‚Äî The TV/Film `#mode-toggle` slider now uses **`$accent` in both states** (TV previously fell through to Textual's off-state `$panel` because only `.-on` was overridden). Focus no longer uses `text-style: underline` (which bisected the slider glyph); switches reserve **`border-bottom: hkey`** (`$boost` blurred, `$accent` focused) so the cue sits strictly below the slider with **no layout jitter**. The second-header bar is **two rows** tall (blurred and focused). Pinned by `tests/unit/test_mode_provider.py::test_mode_toggle_slider_renders_success_in_both_tv_and_film` and `test_bar_height_is_stable_two_rows`. Application **0.7.1** (PATCH); `tui/` module spec **1.6.1**; spec **1.12.1** unchanged.

- **TUI ‚Äî focus-jitter on `ModeProviderBar` toggles** ‚Äî Tab-focusing or clicking a mode / provider `Switch` used to grow the widget from 8√ó1 to 10√ó3 cells (and shift sibling labels) because the stock Textual `Switch` default CSS is `border: tall $border-blurred;` blurred ‚Üí `border: tall $border;` on `:focus`, and `ModeProviderBar`'s previous `Switch { border: none; }` override only reset the blurred state. The widget now reserves the `tall` border in both states (`border: tall transparent;` blurred ‚Üí `border: tall $accent;` on `:focus` + a subtle `background-tint: $foreground 5%;`), matching the same **"reserve the border, only change the color"** pattern used on `#nav_panel` / `#plan_panel`. A defensive `#plan_panel Button:focus, #confirm-bar Button:focus { border: none; }` override was added to [`src/termrenamer/tui/theme.tcss`](src/termrenamer/tui/theme.tcss) so a future Textual release that adds a focus border to the default `Button` cannot reintroduce jitter on Posting-flat buttons. Keyboard focusability (Tab, Enter/Space) is unchanged. Regression pinned by `tests/unit/test_mode_provider.py::test_switch_size_is_stable_across_focus_transitions`. See the `Q5 D ‚Äî Focus-jitter refinement` sub-answer in [`docs/research/2026-04-17-textual-style-borders-trees/research.md`](docs/research/2026-04-17-textual-style-borders-trees/research.md). Application **0.6.1** (PATCH, bug fix); `tui/` module spec **1.5.1**; spec **1.12.1** unchanged (no architectural contract change).

### Fixed

- **TUI ‚Äî log rendering corruption** ‚Äî HTTP log lines (from `httpx` via stdlib logging) were written directly to the real terminal stderr, bypassing Textual's display and rendering on top of UI panels. Root cause: `StreamHandler` captured the original `sys.stderr` reference at bootstrap, before Textual replaced it with `_PrintCapture`. Fix: on mount, the stderr handler is removed and replaced with a `_RichLogHandler` that routes all log output into the **Logs** tab's `RichLog` widget.
- **TUI ‚Äî layout restructure** ‚Äî Replaced the three-panel layout with a split layout: Source (directory browser) and Preview (plan table) are side-by-side on the main screen; Logs are isolated in a bottom tabbed panel. Added a selected-folder indicator below the directory browser. This eliminates rendering contention and keeps high-frequency log output out of the primary workflow.
- **Tests:** `test_missing_tmdb_key_raises` uses an empty `env_file` when calling `load_settings` so `python-dotenv` does not reload `TERMRENAMER_TMDB_API_KEY` from the repository `.env` after `monkeypatch.delenv`, keeping the assertion deterministic on developer machines that use a local `.env`.
